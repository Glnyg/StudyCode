# Git Worktree

## 这是什么
- `Git Worktree` 是 Git 提供的多工作目录能力。
- 你可以把它理解成：同一个仓库，同时开多个独立目录做不同任务。

## 为什么重要
- 它可以让你并行做多个需求，而不用频繁 stash 或来回切分支。
- 对 AI 代理特别有用，因为一个任务可以绑定一个独立上下文目录。

## 在本项目里怎么用
- 主目录通常保留给 `main`。
- Codex 的单独任务会放在独立 worktree 里处理。
- 做完后再从任务分支合回主工作树。

## 工作里怎么用
- 适合并行开发、修 bug、写文档、做实验分支。

## 面试怎么说
- “我会用 Git Worktree 给并行任务隔离工作目录，减少分支切换和 stash 带来的上下文污染。”
