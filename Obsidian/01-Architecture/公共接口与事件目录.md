# 公共接口与事件目录

对应正式文档：`docs/api/public-contracts-and-events.md`

## 这是什么
- 这篇是系统对外和对内沟通的“语言规范”。
- `HTTP/gRPC` 是同步接口。
- 事件是异步通信。

## 核心事件
- `MessageAppended`
- `MessageRedacted`
- `ConversationAssigned`
- `TransferCompleted`
- `AiHandoffChanged`
- `AssetSent`
- `UrgentInterventionTriggered`
- `UrgentInterventionAcknowledged`
- `ManagementNotificationDispatched`

## 核心规则
- 事件必须带 `tenant_id`
- 事件必须可 [[幂等]] 消费
- 改事件契约要改 ADR

## 为什么重要
- 很多微服务系统不是业务逻辑崩，而是“契约漂移”崩。
- 如果字段今天加、明天改、后天删，但没人记，系统很快会变得不可维护。

## 工作里怎么用
- 改接口前先改文档。
- 改事件前先想兼容性。
- 对下游影响大的字段改动要补迁移说明。
- 涉及企微、飞书这种内部通知时，也要把告警事件和补偿语义写清楚。

## 面试怎么说
- “我会把 API 和事件契约当成正式资产管理，而不是聊天里口头约定。这样做能减少微服务协作里的隐形故障。”
